### Quick start **ZUIX** will come handy, either creating a simple site like a blog or a *strongly componentized* [*SPA*](https://en.wikipedia.org/wiki/Single-page_application) for an Home Automation dashboard. If your target is creating something more complex than a blog site, before going into further reading, perhaps it may come useful to read something about the [**MV\***](https://addyosmani.com/resources/essentialjsdesignpatterns/book/#detailmvcmvp) design pattern. Either way, *ZUIX* will intrinsically guide you through adopting such design pattern. The starting point of your application will be an ```.html``` file containing header includes, such as CSS styles, libraries and frameworks used by your application, including *ZUIX*. Beside that, the body of this ```.html``` file, will be blank in most cases, and will only contain some include directives to tell ZUIX what content and components to load in the page. #### Static content and templates So, to include static content or a template, the ```data-ui-include``` attribute is used. The value of this attribute is the URL of the *HTML* file to include omitting its ```.html``` extension. The URL can either refer to a file on the local server or a remote one.
