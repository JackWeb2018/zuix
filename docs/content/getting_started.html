#### NPM / Bower (TODO: add npm and bower) ### Getting started The core functionality of *ZUIX* is the capability of loading contents and components from file. This makes possible creating a page and dynamically loading into it all other contents and components as needed. Using this approach, a page, may consist of an ```.html``` file containing header includes, such as CSS styles, libraries and frameworks, including *ZUIX* and a body that will mostly have only a few **include directives** that tell *ZUIX* what contents and components to load. #### Content inclusion The View So, as shown in the previous example, to load a content from file, the ```data-ui-include``` attribute is used. The value of this attribute is the URL of the HTML file to include without the ```.html``` extension. This kind of URL is the **View** *type identifier*, where the *View* represents the loadable content itself. After the content is loaded, it is appended to element declaring the ```data-ui-include``` attribute, that will then be the container (view container) wrapping the loaded content. The *View type identifier* can either refer to a file on the local server or a remote one. EXAMPLE VIEW - **HTML** file [site/common/links.html](https://genielabs.github.io/zuix/site/common/links.html)

##### Content styling When including a view, after loading the ```.html``` file, *ZUIX* will try to load a ```.css``` style file with the same location/basename. If the style file is found, all style definitions contained in it, will be applied to the container where the view has been loaded. ##### Data binding #### Components A component is basically a [content template](#Content-Templates) (html/css/data model) with the plus of a piece of JavaScript code associated to it. This JavaScript code is the ****Controller**** of the component and it implements the logic and behavior of it upon user interaction or when the bound data model is updated. #### Loading a component To load a component, the ```data-ui-load``` attribute is used, where its value is the name of the view template that has to be applied to the component.
