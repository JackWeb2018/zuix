### zuix API documentation The ```zuix``` global object exposes the following methods: #### ```zuix.load( name [, options] )``` Loads a component and returns the associated ```{ComponentContext}``` object. This method will try in order: - succesfully get the view associated to the component ```name``` or fire the ```options.error``` callback on failure - if previous step was succesfull it will cache the view (other components may refer the same view) - then it will try to load a {ContextController} either from the inline ```options.controller``` or from the ```name```.js 
Method summary
    {ComponentContext} zuix.load( view [, options] )

Parameters
    {string|jQuery|HTMLElement} view
    Mixed type parameter that can be either one of the following:
    - a ```string``` with the value matching the ```data-ui-view```
      attribute of an inline element in the document
    - a ```string``` indicating the relative or absolute path
      of the view file (without the .html extension)
    - a ```{jQuery}``` $(element)
    - a ```HTMLElement```
    {ContextOptions} options
        Options object used to initialize the component context.

Returns
    {ComponentContext}
    The loaded component context.

The {ContextController} function skelton is described below: 

function (cp) {

    cp.create = function () {
        // Called as soon as the component context
        // is created. Add initizlization code
        // and register UI  event handlers here.
    };

    cp.destroy = function() {
        // Called as soon as the component context
        // is destroyed. Dispose any allocated resource here.
    };

    cp.resume = function () {
        // Called automatically after cp.create()
        // and any time back from cp.pause().
    };

    cp.pause = function() {
        // Called when the component is
        // requested to pause. Stop any background
        // operation here
        // (eg. event handlers and background workers)
    };

    cp.refresh = function () {
        // Called when the bound data set (model) has changed.
    };

    cp.event = function (eventPath, eventValue) {
        // Called when a component event is fired.
    };

    cp.api = function(command, options) {
        // Called when a local API command is invoked.
    };

};

cp is the {ComponentContext} object that is passed to the {ContextController} function once the related component is created. When calling the zuix.controller(...) method, the passed {ContextController} function can implement none, one or more of the cp context handlers. ##### cp {ComponentContext} object summary

#### ```zuix.field( name [, container])``` Returns the ```{jQuery}``` object of the element having ```data-ui-field``` attribute matching ```name```. If no ```container``` specified the search range will cover the entire document. 
Method summary
    {jQuery} zuix.field( name [, container] )

Parameters
    {string} name
        ...
    {string|jQuery|HTMLElement} container
        A valid jQuery element/selector.

Returns
    {jQuery}
    The {jQuery} element with ```data-ui-field``` matching ```name```.
